# Phase 1 — UI Design Contract: Foundation + Setup

> Visual and interaction contract for the Setup screen and app-wide theme bootstrap.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Source authority for all locked values: CLAUDE.md (Design System section).
> Do NOT override any locked value without a CLAUDE.md update and explicit sign-off.

---

## Scope

Phase 1 delivers two user-visible surfaces:

1. **App bootstrap** — theme registration, font loading, MaterialApp wrapper, go_router wiring, crash-recovery redirect.
2. **Setup screen** — handicap index input, course search field, search results list, course detail card, missing-rating warning, "Start Round" CTA, API key error state.

Shot Capture and Round Review are out of scope for this phase. Their design contracts are deferred to Phases 2 and 3.

## Color Tokens

Declare as `abstract final class BrdyColors` in `lib/theme/brdy_colors.dart`.

| Token | Hex | Role | 60/30/10 Split | Source |
|-------|-----|------|----------------|--------|
| `background` | `#0A0A0A` | Dominant surface — all screen backgrounds | 60% | CLAUDE.md ("Black / off-white") |
| `surface` | `#1A1A1A` | Secondary surface — cards, input fills, search result tiles | 30% | CLAUDE.md default derived |
| `onSurface` | `#F0F0F0` | Primary text on dark surfaces | — | WCAG AAA on #0A0A0A: ratio ~17:1 |
| `onSurfaceMuted` | `#A0A0A0` | Secondary text (labels, placeholders) | — | Contrast on #0A0A0A: ~7.2:1 (AAA floor) |
| `accent` | `#E8520A` | Orange accent — reserved-for list below | 10% | CLAUDE.md (locked) |
| `onAccent` | `#0A0A0A` | Text/icon on accent-filled buttons | — | Contrast on #E8520A: ~4.8:1 (AA; use only for large button text ≥18dp bold) |
| `destructive` | `#CC2200` | Error states, destructive confirmation copy | — | Distinct from accent; WCAG AA on #0A0A0A |
| `onDestructive` | `#F0F0F0` | Text on destructive surfaces | — | — |
| `divider` | `#2A2A2A` | List separators, card borders | — | — |

**Accent (`#E8520A`) is reserved for:**
- The "START ROUND" primary CTA button background
- The active/focused state border on the course search field
- The course name in a loaded course card (text emphasis only)
- Progress/loading indicators (tile pre-caching progress bar in Phase 1)

Accent must NOT be used on: body text, list item backgrounds, section headers, icon fills (Phase 1 scope). Every other interactive element uses `onSurface` text on `surface` background.

**Contrast compliance (WCAG AAA 7:1 target — CLAUDE.md):**
- `onSurface` (#F0F0F0) on `background` (#0A0A0A): ~17:1 — PASS
- `onSurfaceMuted` (#A0A0A0) on `background` (#0A0A0A): ~7.2:1 — PASS
- `accent` (#E8520A) on `background` (#0A0A0A): ~4.8:1 — FAIL for small text; use only for large elements (buttons ≥18dp bold, course name heading ≥20dp)
- `onSurface` (#F0F0F0) on `surface` (#1A1A1A): ~15:1 — PASS

## Spacing Scale

Flutter `EdgeInsets` / `SizedBox` / `Gap` widget. All values multiples of 4dp.

Declare as `abstract final class BrdySpacing` in `lib/theme/brdy_spacing.dart`.

| Token | dp Value | Usage in Phase 1 |
|-------|----------|-----------------|
| `xs` | 4 | Icon-to-label gap; inline chip padding |
| `sm` | 8 | Input field internal padding top/bottom; list tile vertical padding |
| `md` | 16 | Default horizontal screen padding; element spacing within a card |
| `lg` | 24 | Vertical spacing between major form sections |
| `xl` | 32 | Gap between handicap section and course search section |
| `2xl` | 48 | Spacing above the "START ROUND" CTA |
| `3xl` | 64 | Not used in Phase 1 — reserved for score buttons (Phase 2) |

**Screen horizontal padding:** 16dp on both sides (token: `md`). Applied via `Padding(padding: EdgeInsets.symmetric(horizontal: BrdySpacing.md))` at the Scaffold body level.

**Exceptions:**
- Course search result tiles: 0dp horizontal padding (full-bleed to screen edges with internal 16dp content padding) — creates the brutalist card-stack feel.
- Missing-rating warning banner: 0dp horizontal padding, full-width with 16dp internal padding.

## Typography

Declare as `abstract final class BrdyTextTheme` in `lib/theme/brdy_text_theme.dart`.

Two font families only. Two weights only. Four named styles.

| Style name | Font | Size (dp) | Weight | Line Height | Usage |
|------------|------|-----------|--------|-------------|-------|
| `displayNumeric` | JetBrains Mono | 28 | 700 (Bold) | 1.2 | Handicap index value display; not used in Phase 1 input |
| `bodyNumeric` | JetBrains Mono | 16 | 400 (Regular) | 1.5 | Course rating/slope values in course card |
| `labelLarge` | Barlow Condensed | 18 | 600 (SemiBold) | 1.2 | CTA button text; section headings |
| `bodyLabel` | Barlow Condensed | 14 | 400 (Regular) | 1.5 | Input labels, list tile subtitles, helper text, muted copy |

**Mapping to Flutter `TextTheme`:**

```dart
// lib/theme/brdy_text_theme.dart
static TextTheme get textTheme => TextTheme(
  displaySmall: _jetBrains(28, FontWeight.w700, 1.2),   // displayNumeric
  bodyMedium:   _jetBrains(16, FontWeight.w400, 1.5),   // bodyNumeric
  labelLarge:   _barlow(18, FontWeight.w600, 1.2),       // labelLarge (buttons)
  bodySmall:    _barlow(14, FontWeight.w400, 1.5),       // bodyLabel
);
```

**Handicap index input field:** Barlow Condensed 18dp SemiBold for the entered value (matches `labelLarge`). JetBrains Mono is for displayed numeric output, not input.

**Course name in loaded card:** Barlow Condensed 18dp SemiBold (`labelLarge`) in accent color (`#E8520A`).

**Rating/slope values in loaded card:** JetBrains Mono 16dp Regular (`bodyNumeric`) in `onSurface`.

No other type sizes or weights are permitted in Phase 1.

### 1. Handicap Index Input

**Widget:** `TextFormField` inside a styled container.

| Property | Value |
|----------|-------|
| Label text | `HANDICAP INDEX` |
| Label style | `bodyLabel` (Barlow Condensed 14dp Regular, `onSurfaceMuted`) |

**Validation:** Inline, no modal. If value outside 0.0–54.0: show `bodyLabel` error text in `destructive` color below the field: `"Handicap index must be 0.0 – 54.0"`. Do not block the CTA on invalid HI — warn only.

4. On error: show error snackbar; re-enable button.

## Interaction Contracts

### Crash Recovery Redirect (FOUND-02, FOUND-03)

On app startup, `appStartupProvider` checks Drift for an incomplete round (`completedAt IS NULL`). If found:
- `activeRoundIdProvider` is set to the found round ID.
- go_router redirect fires before Setup renders, pushing user to `/shot-capture/{roundId}`.
- **User never sees the Setup screen** — no loading flash is acceptable. Use a splash/loading widget at the router level for the async check duration.
- Splash widget: `background` color, centered `BRDY.01` wordmark in `labelLarge` style. No spinner.

### Course Pre-Select (SETUP-04)

On Setup screen open (no active round), `hivePlayerPrefsProvider` is read for `last_used_course_id`. If present, `courseRepositoryProvider` loads the cached course from Hive (no network request). The Course Card is pre-populated. The search field remains empty.

### Debounce Pattern

Search field debounce is 400ms. Implementation: use `ref.debounce` or a `Timer` inside the provider notifier. Do not call the API on every keystroke.

### Haptic Feedback

On successful course load (Course Card appears): `HapticFeedback.lightImpact()` via the `haptic_feedback` package (already in pubspec). On START ROUND tap: `HapticFeedback.mediumImpact()`.

### Android Back Button (Setup screen)

No `PopScope` needed — Setup is the root of the stack. Android back exits the app (default behavior). Do not intercept.
