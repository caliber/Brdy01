## Screen Layout

This section is prescriptive ‚Äî executor must follow this arrangement.

### ShotCaptureScreen overall structure

```
Scaffold (background: BrdyColors.background)
‚îî‚îÄ‚îÄ SafeArea
    ‚îî‚îÄ‚îÄ Column (full height, no scroll)
        ‚îú‚îÄ‚îÄ ScoreBar                    [fixed height ~56dp, surface background]
        ‚îú‚îÄ‚îÄ HoleHeader                  [fixed height ~56dp, surface background]
        ‚îú‚îÄ‚îÄ Divider (color: divider)
        ‚îú‚îÄ‚îÄ FairwayGirToggles           [fixed height ~48dp per row, transparent]
        ‚îú‚îÄ‚îÄ PuttsCounter                [fixed height ~64dp, transparent]
        ‚îú‚îÄ‚îÄ Spacer                      [flex ‚Äî pushes outcome buttons to bottom]
        ‚îú‚îÄ‚îÄ OutcomeButtonGrid           [fixed height: 2 rows √ó 80dp = 160dp + BrdySpacing.sm gap]
        ‚îú‚îÄ‚îÄ Divider (color: divider)
        ‚îî‚îÄ‚îÄ HoleNavDrawer               [fixed height ~56dp, surface background]
```

Score bar and hole nav drawer are positioned OUTSIDE the hole content area so they persist visually across hole transitions.

### ScoreBar (SHOT-10)

- Background: `BrdyColors.surface`
- Padding: `EdgeInsets.symmetric(horizontal: BrdySpacing.md, vertical: BrdySpacing.sm)`
- Layout: `Row` with `MainAxisAlignment.spaceBetween`
- Left: "SCORE" in `textTheme.bodySmall` + `BrdyColors.onSurfaceMuted`
- Right: score value in `textTheme.displaySmall`; color = `BrdyColors.accent` if negative, `BrdyColors.onSurface` if zero or positive
- Score format: `E` (even), `+{n}` (over), `-{n}` (under) ‚Äî use ASCII hyphen-minus for under-par, not Unicode minus sign

### HoleHeader (SHOT-11)

- Background: `BrdyColors.surface`
- Padding: `EdgeInsets.symmetric(horizontal: BrdySpacing.md, vertical: BrdySpacing.sm)`
- Layout: `Row` with `MainAxisAlignment.spaceBetween`
- Left: "HOLE {n}" in `textTheme.displaySmall` (JetBrains Mono 28dp bold)
- Center: "PAR {n}" in `textTheme.labelLarge` (Barlow Condensed 18dp bold)
- Right: "SI {n}" or "SI ‚Äì" in `textTheme.bodySmall` + `BrdyColors.onSurfaceMuted`

### OutcomeButtonGrid (SHOT-01, SHOT-02)

Layout: 2-row grid
- Row 1 (3 buttons): BIRDIE | PAR | BOGEY ‚Äî equal flex width
- Row 2 (2 buttons): DOUBLE | PICKUP ‚Äî equal flex width (wider than row 1 buttons individually)
- Gap between rows: `BrdySpacing.sm` (8dp)
- Gap between buttons in a row: `BrdySpacing.xs` (4dp)

Button dimensions:
- Minimum width: 64dp (calculated by `Expanded` flex, will be wider in practice)
- Minimum height: 80dp ‚Äî enforce via `BoxConstraints(minHeight: 80)`
- Border radius: 0dp (brutalist ‚Äî `RoundedRectangleBorder()`)
- Text style: `textTheme.labelLarge` (Barlow Condensed 18dp bold)

Button colors:

| Button | Background | Foreground |
|--------|-----------|-----------|
| BIRDIE | `BrdyColors.accent` | `BrdyColors.onAccent` |
| PAR | `BrdyColors.surface` | `BrdyColors.onSurface` |
| BOGEY | `BrdyColors.surface` | `BrdyColors.onSurface` |
| DOUBLE | `BrdyColors.destructive` | `BrdyColors.onDestructive` |
| PICKUP | `BrdyColors.destructive` | `BrdyColors.onDestructive` |

BIRDIE button uses `GestureDetector` wrapping (not `InkWell`) to support `onDoubleTap` for EAGLE. All other buttons use `InkWell`. EAGLE outcome is not displayed as a distinct button ‚Äî it is triggered by double-tapping BIRDIE only.

Animation on tap: `flutter_animate` `.animate().scale(begin: Offset(1.0,1.0), end: Offset(0.96, 0.96), duration: 80ms).then().scale(end: Offset(1.0, 1.0), duration: 80ms)` ‚Äî brief press-and-release scale. Apply to the button container.

Haptic feedback: `HapticFeedback.mediumImpact()` on every outcome tap (including EAGLE double-tap).

### PuttsCounter (SHOT-05)

- Layout: `Row` centered horizontally
- Left: `IconButton(Icons.remove, iconSize: 32)` ‚Äî disabled (grayed) when putts == 0
- Center: `SizedBox(width: 64)` containing putts count in `textTheme.displaySmall` centered
- Right: `IconButton(Icons.add, iconSize: 32)`
- Label above: "PUTTS" in `textTheme.bodySmall` + `BrdyColors.onSurfaceMuted`, centered
- Padding: `EdgeInsets.symmetric(vertical: BrdySpacing.sm)`

### FairwayGirToggles (SHOT-06, SHOT-07)

- Layout: `Column` of toggle rows
- Fairway row: shown only when `holePar != 3` ‚Äî completely absent from widget tree on par 3, not just hidden
- GIR row: always shown
- Each toggle row:
  - Layout: `Row` with `MainAxisAlignment.spaceBetween`, padding `EdgeInsets.symmetric(horizontal: BrdySpacing.md, vertical: BrdySpacing.xs)`
  - Left: label ("FAIRWAY HIT" or "GIR") in `textTheme.bodySmall` + `BrdyColors.onSurfaceMuted`
  - Right: two-state toggle button ‚Äî "HIT" / "MISS" (fairway) or "GIR" / "NO GIR"
  - Active/hit state: background `BrdyColors.accent`, text `BrdyColors.onAccent`
  - Inactive/miss state: background `BrdyColors.surface`, text `BrdyColors.onSurface`
  - Minimum tap target: 64dp wide √ó 48dp tall

### HoleNavDrawer (SHOT-12)

- Background: `BrdyColors.surface`
- Height: 56dp
- Layout: horizontal `ListView` of 18 `HoleChip` widgets with `scrollDirection: Axis.horizontal`
- Padding: `EdgeInsets.symmetric(horizontal: BrdySpacing.sm, vertical: BrdySpacing.sm)`

Each HoleChip:
- Width: 40dp, Height: 40dp
- Border radius: 0dp (brutalist)
- Label: hole number (1‚Äì18) in `textTheme.bodySmall`
- Unscored: background `BrdyColors.surface`, border `BorderSide(color: BrdyColors.divider, width: 1)`
- Scored with outcome: background = outcome color (see Outcome color coding table above)
- Active hole (currently displayed): border `BorderSide(color: BrdyColors.onSurface, width: 2)`
- Future hole (beyond highest scored + 1): opacity 0.4, `onTap: null` (not tappable)
- Gap between chips: `BrdySpacing.xs` (4dp)

"BACK TO CURRENT" affordance: visible only when `activeHoleIndex < highestScoredHoleIndex`. Shown as a small text button (`textTheme.bodySmall`, color `BrdyColors.accent`) at the far right of the nav strip, always in view (not scrolled away). Implementation: place in a `Row` alongside the `Expanded` `ListView`, separated by `BrdySpacing.sm`.

## Interaction Contracts

### Outcome tap flow (SHOT-01, SHOT-03, SHOT-04)

1. User taps outcome button
2. Scale animation fires immediately (80ms press + 80ms release)
3. Haptic: `HapticFeedback.mediumImpact()`
4. `HoleScoreNotifier.recordOutcome()` is called ‚Äî Drift write, then `activeHoleIndexProvider` advances
5. Undo SnackBar appears (see below)
6. ScoreBar and HoleHeader both rebuild reactively from providers

### EAGLE double-tap (SHOT-02)

- First tap on BIRDIE: 300ms wait for double-tap window (Flutter `kDoubleTapTimeout`)
- If second tap arrives within 300ms: EAGLE outcome recorded, animation fires
- If no second tap: BIRDIE outcome recorded after 300ms delay
- This delay is accepted for this app context. No workaround required.
- On EAGLE: scale animation + `HapticFeedback.mediumImpact()`

### Undo toast (SHOT-04)

- Appears immediately after outcome write completes
- SnackBar content: "{OUTCOME} ‚Äî Hole {n}" in `textTheme.bodySmall` + `BrdyColors.onSurface`
- SnackBar background: `BrdyColors.surface`
- SnackBar action label: "UNDO" in `BrdyColors.accent`
- Duration: 4 seconds (`Duration(seconds: 4)`)
- `persist: false` ‚Äî required for auto-dismiss with action in Flutter 3.38+
- On UNDO tap: `HapticFeedback.lightImpact()`, then undo logic executes
- At most one toast visible at a time ‚Äî call `ScaffoldMessenger.hideCurrentSnackBar()` before showing new one

### Putts counter (SHOT-05)

- Every `+` or `-` tap writes to Drift immediately (no debounce)
- Minimum value: 0 (minus button disabled, not absent, when putts == 0)
- No maximum enforced (no upper clamp)
- No haptic on putts counter (low-priority action; avoid feedback noise)

### Fairway / GIR toggles (SHOT-06, SHOT-07)

- Tap toggles between hit and miss states
- Write to Drift immediately on every toggle
- Fairway toggle: writes `null` (not `false`) for par 3 holes ‚Äî enforced at the notifier level, not the widget level; widget is absent on par 3 anyway
- No animation beyond Material state ripple

### Hole navigation (SHOT-12)

- Tap a scored HoleChip: sets `activeHoleIndexProvider` to that index (internal state, no route push)
- "BACK TO CURRENT" tap: sets `activeHoleIndexProvider` to `highestScoredHoleIndex`
- No transition animation between holes (instant state swap ‚Äî brutalist)
- Current hole content updates synchronously as providers rebuild

### Round completion

- After hole 18 outcome is recorded and `roundCompleteProvider` emits `true`
- Screen calls `context.go('/round-review/$roundId')` via `ref.listen`
- No confirmation dialog ‚Äî round completes immediately
- `activeHoleIndexProvider` is reset to 0 on new round start (not relevant to this screen but noted for cross-cutting correctness)
