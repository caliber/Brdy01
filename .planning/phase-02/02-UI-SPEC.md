>

---

## Scope

Phase 2 delivers one screen: **ShotCaptureScreen** — the full hole-by-hole scoring interface.

The screen has two visually distinct zones separated by a 1dp horizontal divider:

1. **Top zone** (~55% of screen height) — `BrdyColors.background` (#0A0A0A). Displays course context, hole navigation, the giant hole number with running score badge, and the SHOTS total.
2. **Bottom zone** (~45% of screen height) — `BrdyColors.onSurface` (#F0F0F0) light grey. Contains the BRDY.01 wordmark rule, two rows of outcome buttons with dot indicators, pagination dots, and the bottom toggle strip.

Setup and Round Review are out of scope for this phase.

### 2. Giant Hole Number + Score Badge (Top Zone Centre)

Position: centre of top zone. `Row(mainAxisAlignment: MainAxisAlignment.center)`.

| Element | Widget | Spec |
|---------|--------|------|
| Left chevron | `IconButton` | `Icons.chevron_left` 32dp `BrdyColors.onSurface`; 48×48dp tap target; disabled → `BrdyColors.onSurfaceMuted`, `onPressed: null`; disabled when `activeHoleIndex == 0` |

### 3. SHOTS Total (Top Zone Footer)

Position: horizontally centred, below the hole number row. `BrdySpacing.md` gap above.

| `SHOTS {n}` | Barlow Condensed 14dp w700 `BrdyColors.onSurface`; `n` = total gross strokes across all scored holes |

### 5. Outcome Button Grid — Row 1 and Row 2

Two rows of 4 buttons each. Each button has a dot indicator and an abbreviated label sitting above it.

**Above-button structure (top to bottom for each button column):**
1. Abbreviated label — Barlow Condensed 10dp w700 `#4A4A4A` ALL CAPS; centered
2. `SizedBox(height: BrdySpacing.xs)` — 4dp
3. Dot indicator — 6dp diameter `Container` with `BoxDecoration(shape: BoxShape.circle)`; filled when outcome matches, outlined (1dp border, transparent fill) when inactive
4. `SizedBox(height: BrdySpacing.xs)` — 4dp
5. Button itself

**Row 1 — 4 columns:**

| Position | Button label | Abbrev | Active fill | Active text | Inactive fill | Inactive text | Dot active color |
|----------|-------------|--------|-------------|-------------|---------------|---------------|-----------------|
| 1 | `PAR` | `PAR` | `Color(0xFF2563EB)` | `#FFFFFF` | `BrdyColors.surface` #1A1A1A | `BrdyColors.onSurface` | `Color(0xFF2563EB)` |
| 2 | `SUB` | `SUB` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.onSurfaceMuted` |
| 3 | `ADD` | `ADD` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.onSurfaceMuted` |
| 4 | `BIRDY` | `BIR` | `BrdyColors.accent` | `BrdyColors.onAccent` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.accent` |

**Row 1 notes:**
- SUB and ADD are putts controls embedded in the button row — they do not record an outcome. They have no persistent active state; they show a momentary tap feedback only.
- PAR shows "active" state when the current hole's recorded outcome is `HoleOutcome.par`.
- BIRDY shows "active" when outcome is `birdie`. When outcome is `eagle`, BIRDY button transforms: fill becomes `Color(0xFFFFD700)`, text becomes `BrdyColors.background`, abbreviated label changes to `EGL`, dot fills gold.

**Row 2 — 4 columns:**

| Position | Button label | Abbrev | Active fill | Active text | Inactive fill | Inactive text | Dot active color |
|----------|-------------|--------|-------------|-------------|---------------|---------------|-----------------|
| 1 | `PICKUP` | `PKU` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.onSurfaceMuted` |
| 2 | `BOGEY` | `BGY` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.onSurfaceMuted` |
| 3 | `DOUBLE` | `DBL` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.surface` | `BrdyColors.onSurface` | `BrdyColors.onSurfaceMuted` |
| 4 | `NEXT` | `HLE` | `BrdyColors.accent` | `BrdyColors.onAccent` | `BrdyColors.accent` (dimmed: opacity 0.4) | `BrdyColors.onAccent` | — (no dot concept for NEXT) |

**Row 2 notes:**
- PICKUP, BOGEY, DOUBLE show active state when the current hole's outcome matches.
- NEXT is a navigation button, not an outcome button. It is always accent-colored. When no outcome is recorded for the current hole, NEXT dims to accent at 40% opacity — it is still rendered but appears unavailable (no `onPressed: null` needed; the notifier ignores the tap if no outcome set, OR disable the button: `onPressed: hasOutcome ? _advance : null`). Preferred: disable `onPressed` to null when no outcome — cleaner feedback.
- NEXT does NOT record any score. It only calls `activeHoleIndexProvider.notifier.set(holeIndex + 1)`.

**Button style (all outcome buttons and NEXT):**
- `ElevatedButton` or custom `InkWell`-wrapped `Container`
- `BoxDecoration(borderRadius: BorderRadius.circular(8), boxShadow: [BoxShadow(color: Colors.black.withOpacity(0.4), blurRadius: 4, offset: Offset(0, 2))])`
- Text: Barlow Condensed 13dp w700 ALL CAPS
- Width: equal `Expanded` in `Row`
- Height: minimum 80dp — `BoxConstraints(minHeight: 80)`
- Border radius: 8dp (per design reference — rounded corners with 3D shadow depth)

**IMPORTANT — BIRDIE uses `GestureDetector`, all others use `InkWell`:**
```dart
// BIRDIE only:
GestureDetector(
  onTap: () => _recordOutcome(HoleOutcome.birdie),
  onDoubleTap: () => _recordOutcome(HoleOutcome.eagle),
  child: _OutcomeButtonBody(label: 'BIRDY', ...),
)

// All others:
InkWell(
  onTap: () => _recordOutcome(HoleOutcome.bogey),
  child: _OutcomeButtonBody(label: 'BOGEY', ...),
)
```

**Animation on outcome tap:** `flutter_animate` `.animate().scale(begin: const Offset(1, 1), end: const Offset(0.95, 0.95), duration: 80ms).then().scale(end: const Offset(1, 1), duration: 80ms)` applied to the button `Container`.

**Gap between rows:** `BrdySpacing.sm` (8dp).

**Gap between buttons in a row:** `BrdySpacing.xs` (4dp).

**Layout structure:**
```dart
Column(children: [
  _ButtonRowWithIndicators(buttons: [PAR, SUB, ADD, BIRDY]),
  SizedBox(height: BrdySpacing.sm),
  _ButtonRowWithIndicators(buttons: [PICKUP, BOGEY, DOUBLE, NEXT]),
```

### 7. Bottom Toggle Strip

Three equal-width toggle buttons. Each has a label on top (coloured by state) and a static descriptor below.

**Structure per toggle (top to bottom):**
1. State label — Barlow Condensed 12dp w700 ALL CAPS; active=`BrdyColors.background` on accent fill, inactive=`#4A4A4A` on light fill
2. Button background fill
3. Descriptor — Barlow Condensed 10dp w400 ALL CAPS `#4A4A4A`; sits below the button

Actually per design reference the label and descriptor are both within the button boundary:

**Button interior (vertical stack):**
- Top: state label (12dp w700)
- Bottom: descriptor (10dp w400)

| Toggle | Active label | Inactive label | Descriptor | Active fill | Inactive fill |
|--------|-------------|----------------|------------|-------------|---------------|
| Fairway hit | `HIT` | `MISS` | `FAIRWAY` | `BrdyColors.accent` #E8520A | `Color(0xFFD0D0D0)` |
| GIR | `GREEN` | `MISS` | `REG` | `BrdyColors.background` #0A0A0A | `Color(0xFFD0D0D0)` |
| Voice | `ON` | `OFF` | `VOICE` | `BrdyColors.background` #0A0A0A | `Color(0xFFD0D0D0)` |

- Active label color on accent fill: `BrdyColors.onAccent` #0A0A0A
- Active label color on dark (#0A0A0A) fill: `BrdyColors.onSurface` #F0F0F0
- Inactive label color on `#D0D0D0` fill: `BrdyColors.background` #0A0A0A
- Descriptor color: `BrdyColors.background` #0A0A0A in all states (on light bg bottom zone)

**Button style:**
- Border radius: 4dp
- Box shadow: `BoxShadow(color: Colors.black.withOpacity(0.3), blurRadius: 3, offset: Offset(0, 1))`
- Width: `Expanded` (equal thirds of available row width)
- Height: minimum 48dp

**FAIRWAY toggle — par 3 gate (SHOT-06):** When `holePar == 3`, the FAIRWAY toggle widget is completely absent from the widget tree (`if (holePar != 3)`). The remaining two toggles (GIR and VOICE) fill the 3-column space equally:
```dart
Row(children: [
  if (holePar != 3) Expanded(child: _FairwayToggle()),
  if (holePar != 3) SizedBox(width: BrdySpacing.sm),
  Expanded(child: _GirToggle()),
  SizedBox(width: BrdySpacing.sm),
  Expanded(child: _VoiceToggle()),
])
```

**VOICE toggle (Phase 2):** Always rendered as inactive (`OFF` / `VOICE`). `onTap: null` — no logic implemented. Not disabled-looking — it simply does nothing when tapped. Phase 5 implements voice.

A horizontal scrollable row of 18 hole chips, accessible by tapping the giant hole number in the top zone (or tapping a dedicated navigation icon if one is added). Slides in from below the hole number row via `flutter_animate` height animation.

## Interaction Contracts

### Outcome Recording (SHOT-01, SHOT-03, SHOT-04)

1. User taps an outcome button (PAR, BOGEY, DOUBLE, PICKUP) or taps BIRDIE (single = birdie, double = eagle).
2. Scale animation fires immediately on the button (80ms press + 80ms release).
3. `HapticFeedback.mediumImpact()` fires.
4. `HoleScoreNotifier(roundId, holeIndex).recordOutcome(outcome)` writes to Drift.
5. `activeHoleIndexProvider.notifier.set(holeIndex + 1)` fires (or `roundCompleteProvider` triggers if hole 18).
6. `ScaffoldMessenger.showSnackBar(undoToast)` fires.
7. `runningScoreProvider` and related providers rebuild reactively from the Drift stream.

### EAGLE Double-Tap (SHOT-02)

- `GestureDetector.onDoubleTap` on BIRDIE button only.
- Single tap: 300ms delay (Flutter `kDoubleTapTimeout`) then records `HoleOutcome.birdie`.
- Double tap: records `HoleOutcome.eagle` immediately.
- 300ms delay is accepted for this app context.
- On eagle: button fill transitions to gold via `flutter_animate` color tween (150ms), abbreviated label changes to `EGL`.

### NEXT Button

- Disabled (`onPressed: null`, opacity 0.4) when current hole has no recorded outcome.
- Enabled when current hole has any recorded outcome.
- On tap: `activeHoleIndexProvider.notifier.set(holeIndex + 1)` — no score write, no undo toast.
- `HapticFeedback.lightImpact()` on NEXT tap.

### Undo (SHOT-04)

- On UNDO tap: `HapticFeedback.lightImpact()`.
- `HoleScoreNotifier(roundId, lastHoleIndex).undoOutcome()` — clears `outcome` field in Drift, sets `activeHoleIndexProvider` back to `lastHoleIndex`.
- Only the immediately preceding outcome is undoable. Toast auto-dismisses at 4 seconds, removing the undo window.

### Putts Counter — SUB / ADD (SHOT-05)

- SUB tap: decrements putts by 1. `onPressed: null` when putts == 0 — button still renders, just disabled.
- ADD tap: increments putts by 1. No maximum.
- Every tap: immediate `HoleScoreNotifier(roundId, holeIndex).setPutts(newValue)` → Drift write.
- `HapticFeedback.selectionClick()` on each tap.

### Fairway and GIR Toggles (SHOT-06, SHOT-07)

- Tap toggles active/inactive state.
- Immediate Drift write: `HoleScoreNotifier.setFairwayHit(bool?)` and `setGir(bool?)`.
- Fairway toggle absent on par 3 — gate is in widget tree, not in notifier.
- `HapticFeedback.selectionClick()` on each toggle tap.
- `fairwayHit: null` (not `false`) written for par 3 holes at outcome record time — notifier responsibility.

### Hole Back-Navigation (SHOT-12)

- Tap giant hole number → hole navigation strip slides in.
- Tap a previously scored chip → `activeHoleIndexProvider.notifier.set(targetIndex)`.
- Tap NOW chip → `activeHoleIndexProvider.notifier.set(highestScoredHoleIndex)`.
- No GoRouter push. No back-stack growth.
- `HapticFeedback.lightImpact()` on chip tap.

### Running Score (SHOT-10)

- `runningScoreProvider(roundId)` derives from `holeListProvider(roundId)` (Drift `watchHolesForRound` stream).
- Running score badge rebuilds on every Drift write.
- Under par: badge text = `-{n}`, badge background = `BrdyColors.accent`.
- Even par: badge text = `E`, badge background = `BrdyColors.accent`.
- Over par: badge text = `+{n}`, badge background = `BrdyColors.accent`.
- Badge background is always accent — score direction does not change badge color. Score text is always `BrdyColors.onAccent`.

### Round Completion

- After hole 18 outcome recorded, `roundCompleteProvider(roundId)` emits `true`.
- `ref.listen` in `ShotCaptureScreen` calls `context.go('/round-review/$roundId')`.
- No confirmation dialog.
- `activeHoleIndexProvider` reset to 0 occurs in `RoundSetupNotifier.createRound()`, not here.

### Haptic Feedback Summary

| Action | Haptic |
|--------|--------|
| Outcome button tap (PAR / BOGEY / DOUBLE / PICKUP / BIRDIE / EAGLE) | `HapticFeedback.mediumImpact()` |
| NEXT tap | `HapticFeedback.lightImpact()` |
| UNDO tap | `HapticFeedback.lightImpact()` |
| Putts +/− tap | `HapticFeedback.selectionClick()` |
| Toggle tap (fairway/GIR) | `HapticFeedback.selectionClick()` |
| Hole chip tap | `HapticFeedback.lightImpact()` |

## Animation Contract

All animations use `flutter_animate`. Brutalist aesthetic — keep micro, never decorative.

| Element | Animation | Duration | Curve |
|---------|-----------|----------|-------|
| Outcome button press | Scale 1.0→0.95→1.0 | 160ms total (80+80) | `Curves.easeInOut` |
| Hole number change | Fade out (50ms) + fade in (100ms) with new number | 150ms | `Curves.easeOut` |
| Running score badge update | Scale pulse 1.0→1.1→1.0 | 150ms | `Curves.easeInOut` |
| BIRDIE→EAGLE fill change | Color tween accent→gold | 150ms | `Curves.easeIn` |
| Hole navigation strip show/hide | Height 0→56dp or reverse | 200ms | `Curves.easeOut` |
| Undo toast entry | Default `SnackBar` animation | Flutter default | — |

No parallax, no page transitions between holes, no hero animations.

**Empty state (round just started, no holes scored):**
- Running score badge: `E`
- SHOTS label: `SHOTS 0`
- Hole nav strip: all 18 chips in unscored state
- NEXT button: disabled (no outcome recorded)

**Error state (course data unavailable):**
- Course info row: `— PAR —` and `HCP —`
- Hole number: displays normally (from `activeHoleIndexProvider`)
- Outcome buttons: remain fully functional
- No error modal — graceful degradation only

**No destructive confirmation dialogs.** PICKUP is an outcome like any other — the 4-second undo window is the only safety net.
