> Visual and interaction contract for the Round Review screen.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Scope

Phase 3 delivers one screen: **RoundReviewScreen** — navigated to from ShotCaptureScreen via
`context.go('/round-review/$roundId')` after all 18 holes are scored (or when the user manually
navigates to review).

The screen has four content areas rendered in a single scrollable column:

1. **Scorecard grid** (REV-01, REV-02) — 18-hole colour-coded grid, front/back subtotals
2. **Stat cards** (REV-03) — strokes, outcome counts, putts, GIR, fairways
3. **WHS differential panel** (REV-04, REV-05) — calculated or indicative or N/A
4. **Action row** (REV-06, REV-07) — share scorecard + start new round

SHOT-13 (Voice unavailable indicator) lives in ShotCaptureScreen — NOT in scope here.

```dart
Stack(
  clipBehavior: Clip.none,
  children: [
    Text('7', style: ...),
    Positioned(bottom: -3, left: 0, right: 0, child: Container(height: 1, color: BrdyColors.onDestructive)),
    Positioned(bottom: -6, left: 0, right: 0, child: Container(height: 1, color: BrdyColors.onDestructive)),
  ],
)
```

### 3. Scorecard Grid

Two instances: Front 9 (holes 1–9) and Back 9 (holes 10–18). Each is a fixed-layout `Table` or manual `Row`/`Column` grid.

**Grid structure (5 rows × 11 columns for each half):**

| Row | Label col | Holes 1–9 | Subtotal col |
|-----|-----------|-----------|--------------|
| Row A (header) | empty | `01`–`09` (or `10`–`18`) | `F9` (or `B9`) |
| Row B (par) | `PAR` | par values | total par for 9 |
| Row C (score) | `SCR` | coloured score cells | subtotal strokes |
| Row D (putts) | `PTT` | putts values | total putts for 9 |

**Column widths:**
- Label column: 32dp fixed
- Each hole column: `Expanded` (equal share of remaining width after label and subtotal cols)
- Subtotal column: 40dp fixed

**Row heights:** 22dp for header row; 20dp for par row; 28dp for score row; 20dp for putts row.

**Label column (`PAR` / `SCR` / `PTT`):**
- Barlow Condensed 11dp w700 `BrdyColors.onSurfaceMuted`
- `CrossAxisAlignment.center` vertical

**Header row (hole numbers):**
- Barlow Condensed 11dp w700 `BrdyColors.onSurfaceMuted`
- Centered in each cell
- `F9` / `B9` label: Barlow Condensed 11dp w700 `BrdyColors.onSurface`

**Par row:**
- Barlow Condensed 11dp w400 `BrdyColors.onSurfaceMuted`
- Centered in each cell
- Subtotal: same style, `BrdyColors.onSurface`

**Score row (REV-01):**
Each cell is a `Container` with outcome-specific fill and text — see colour table above.
- Scored cell: 28dp tall, centered JetBrains Mono 13dp w700; outcome fill + text per colour table
- Unscored cell: `BrdyColors.divider` fill, `—` glyph in `BrdyColors.onSurfaceMuted` Barlow Condensed 11dp w400
- Subtotal cell: `BrdyColors.surface` fill, JetBrains Mono 13dp w700 `BrdyColors.onSurface`

**Putts row:**
- Barlow Condensed 11dp w400 `BrdyColors.onSurface`
- Centered; `—` if no putts recorded for that hole
- Subtotal: same style

**Cell borders:** `Border.all(color: BrdyColors.divider, width: 0.5)` on every cell. No rounded corners (brutalist).

**Front 9 / Back 9 label (REV-02):**
- `F9` column header label sits in the subtotal column of the Front 9 grid
- `B9` column header label sits in the subtotal column of the Back 9 grid

**Implementation note:** Use `Table` widget with `TableBorder.all(color: BrdyColors.divider, width: 0.5)` for the grid. Set `columnWidths` with `FixedColumnWidth` for label and subtotal columns, `FlexColumnWidth(1)` for hole columns.

### 4. Stat Cards (REV-03)

Stats are grouped into three visual clusters, each rendered as a horizontally-scrollable or wrapping row of equal-width cards:

**Cluster A — Primary scores (3 cards, equal width in `Row`):**

| Card | Label | Value source | Value format |
|------|-------|-------------|--------------|
| STROKES | `STROKES` | `sum(par + outcome_offset)` per scored hole | Integer e.g. `78` |
| VS PAR | `VS PAR` | strokes − total course par | `+{n}` / `E` / `−{n}` |
| NET | `NET SCORE` | gross strokes − playing handicap | Integer e.g. `61` |

Playing handicap formula (for display only): `round(handicapIndex × (slope / 113) + (courseRating − par))`. Net score = gross − playing handicap.

**Cluster B — Outcome counts (7 cards, wrapping `Wrap` with `runSpacing: BrdySpacing.sm`):**

| Card | Label | Value source |
|------|-------|-------------|
| EAGLES | `EAGLES` | count of `eagle` outcomes |
| BIRDIES | `BIRDIES` | count of `birdie` outcomes |
| PARS | `PARS` | count of `par` outcomes |
| BOGEYS | `BOGEYS` | count of `bogey` outcomes |
| DOUBLES | `DOUBLES` | count of `doubleBogey` outcomes |
| TRIPLES | `TRIPLES` | count of holes where `par + outcome_offset >= par + 3` (i.e. triple or worse, excluding pickup) |
| PICKUPS | `PICKUPS` | count of `pickup` outcomes |

**Cluster C — Putting / GIR / Fairway stats (6 cards, wrapping `Wrap`):**

| Card | Label | Value source | Value format |
|------|-------|-------------|--------------|
| PUTTS | `PUTTS` | `sum(putts)` across all scored holes | Integer |
| PT/GIR | `PT/GIR` | total putts ÷ GIR count | 1dp decimal e.g. `1.8`; `—` if GIR count = 0 |
| GIRS | `GIRS` | count of holes where `gir == true` | Integer |
| GIR% | `GIR%` | `(GIR count / scored holes) × 100` | Integer `%` e.g. `61%` |
| FAIRWAYS | `FAIRWAYS` | count of holes where `fairwayHit == true` | Integer |
| FIR% | `FIR%` | `(fairways hit / eligible holes) × 100` (eligible = non-par-3 holes scored) | Integer `%` e.g. `67%` |

**Stat card visual spec:**
- Background: `BrdyColors.surface` #1A1A1A
- Border: `Border.all(color: BrdyColors.divider, width: 1)`
- Border radius: 0dp (brutalist)
- Padding: `EdgeInsets.symmetric(horizontal: BrdySpacing.sm, vertical: BrdySpacing.sm)`
- Minimum width: 80dp; expand via `Wrap` spacing
- Label: Barlow Condensed 11dp w700 `BrdyColors.onSurfaceMuted` ALL CAPS; top of card
- Value (primary): JetBrains Mono 20dp w700 `BrdyColors.onSurface`; bottom of card
- Value (secondary — for smaller counts like TRIPLES/PICKUPS shown at same size): JetBrains Mono 20dp w700

**Eagle stat card exception:** Eagle count value colored `Color(0xFFFFD700)` (gold) not `onSurface` — visual echo of scorecard cell.

**Birdie stat card exception:** Birdie count value colored `BrdyColors.accent` — visual echo.

**VS PAR card — color logic:**
- Under par (negative): value colored `BrdyColors.accent` (orange)
- Even par (`E`): value colored `BrdyColors.onSurface`
- Over par (positive): value colored `BrdyColors.onSurface`

**Cluster separation:** `SizedBox(height: BrdySpacing.sm)` between each cluster.

### 5. WHS Differential Panel (REV-04, REV-05)

A single `Container` card spanning full content width.

**Card spec:**
- Background: `BrdyColors.surface` #1A1A1A
- Border: `Border.all(color: BrdyColors.divider, width: 1)`
- Border radius: 0dp
- Padding: `EdgeInsets.all(BrdySpacing.md)`

**Three display states:**

**State 1 — Complete 18-hole round, Rating+Slope present:**
```
WHS DIFFERENTIAL
+8.4
COURSE RATING 74.0  ·  SLOPE 145  ·  ADJ. GROSS 82
```
- Label `WHS DIFFERENTIAL`: Barlow Condensed 11dp w700 `BrdyColors.onSurfaceMuted`
- Value (e.g. `+8.4`): JetBrains Mono 28dp w700 `BrdyColors.onSurface`; format to 1dp always
- Sub-line: Barlow Condensed 11dp w400 `BrdyColors.onSurfaceMuted`; format `COURSE RATING {value}  ·  SLOPE {value}  ·  ADJ. GROSS {value}`

**State 2 — Fewer than 18 holes recorded (REV-05):**

```
│ WHS DIFFERENTIAL                                  │
│ +6.1                                              │
│ COURSE RATING 74.0  ·  SLOPE 145  ·  ADJ. GROSS 70│
└───────────────────────────────────────────────────┘
```
- Add a `Container(width: 3, color: BrdyColors.destructive)` left-edge rule inside the card
- `INDICATIVE — FEWER THAN 18 HOLES`: Barlow Condensed 11dp w700 `BrdyColors.destructive`; sits above the label+value block
- Differential value displayed in `BrdyColors.onSurface` (not muted — the warning label carries the caution signal)
- Card border changes: `Border.all(color: BrdyColors.destructive, width: 1)` for the indicative state

**State 3 — Rating or Slope missing:**
```
WHS DIFFERENTIAL
N/A
COURSE RATING OR SLOPE NOT AVAILABLE
```
- `N/A`: JetBrains Mono 28dp w700 `BrdyColors.onSurfaceMuted`
- Sub-line: Barlow Condensed 11dp w400 `BrdyColors.onSurfaceMuted`

**Formula for display (REV-04):**
```
Playing Handicap = round(handicapIndex × (slope / 113) + (courseRating − par))
AGS = sum of min(holeScore, par + 2 + handicapStrokesOnHole) per hole
  where holeScore for pickup = par + 2 + handicapStrokesOnHole (net double bogey)
Differential = (AGS − courseRating) × 113 / slope
```
Display differential to 1 decimal place. Positive values shown as `+{n}` (e.g. `+8.4`). Negative shown as `−{n}` using minus sign (U+2212), not ASCII hyphen.

**Handicap strokes on hole:** Requires `strokeIndex` per hole from the course model. `handicapStrokesOnHole` for a given hole = `floor(playingHandicap / 18) + (1 if hole strokeIndex <= playingHandicap mod 18, else 0)`.

If `strokeIndex` is null for any hole: treat as `par + 2` cap (net double bogey without stroke allocation) — safe fallback that slightly understates the cap.

2. Show loading state: replace button label with a 16dp `CircularProgressIndicator` in `BrdyColors.onAccent`.

## Interaction Contracts

### Screen Entry

`RoundReviewScreen` receives `roundId` from the route parameter. Entry is via `context.go('/round-review/$roundId')` from `ShotCaptureScreen`. No back navigation is available (go_router stack is replaced). The Android back button will exit the app from this screen — no `PopScope` interception needed.

### Data Loading

All stats, scorecard, and differential data are derived from Drift via providers. The screen uses `AsyncValue` watch pattern:

```dart
final holes = ref.watch(holeListProvider(roundId));
final course = ref.watch(courseForRoundProvider(roundId));
```

**Loading state:** While `holes` or `course` is `AsyncLoading`, show the screen header with a centered `CircularProgressIndicator(color: BrdyColors.accent)` in the scroll area. No skeleton screens.

**Error state:** If `holes` or `course` is `AsyncError`, show a centred error panel (see Copywriting Contract — Data load error).

**Empty scorecard holes:** Holes that have not been scored (outcome is null) render as unscored cells (`—`). The screen is accessible for review at any time during the round if navigated to manually — it does not require all 18 holes to be complete to render.

### Share Interaction

See Action Row section above. The `share_plus` package is already declared in `pubspec.yaml` (verify before Phase 3 plan). If `share_plus` is absent from `pubspec.yaml`, it must be added as part of the Phase 3 plan task.
