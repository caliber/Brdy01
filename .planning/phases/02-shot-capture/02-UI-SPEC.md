## Spacing Scale

All values are pre-declared in `lib/theme/brdy_spacing.dart`. Use token names ‚Äî never raw numbers.

| Token | Value | Usage in Phase 2 |
|-------|-------|------------------|
| `BrdySpacing.xs` | 4dp | Icon-to-label gaps, internal chip padding |
| `BrdySpacing.sm` | 8dp | Gap between rows in OutcomeButtonGrid, between putts +/- and counter |
| `BrdySpacing.md` | 16dp | Horizontal screen padding, section internal padding |
| `BrdySpacing.lg` | 24dp | Vertical spacing between secondary controls (Putts / Fairway / GIR) |
| `BrdySpacing.xl` | 32dp | Not used in Phase 2 |
| `BrdySpacing.x2l` | 48dp | Not used in Phase 2 |
| `BrdySpacing.x3l` | 64dp | Minimum tap-target height ‚Äî outcome button `BoxConstraints(minHeight: 64)` |

**Exception ‚Äî outcome button tap targets:** CLAUDE.md mandates 64√ó80dp minimum for score buttons (golf gloves, wet hands). The height constraint is `minHeight: 80dp` per button row. The width constraint is `Expanded` (fills available horizontal space in each row). The `BrdySpacing.x3l` token value (64dp) is the floor; the actual rendered height is 80dp.

**Exception ‚Äî putts +/‚àí buttons:** Minimum tap target 48√ó48dp. Not score buttons, but still on-course touch targets.

**Exception ‚Äî hole nav chips:** Minimum tap target 44√ó44dp. Small but infrequently tapped; golfer uses nav strip between holes, not mid-swing.

`ShotCaptureScreen` is a `ConsumerStatefulWidget`. It uses a full-height `Column` inside `SafeArea` ‚Äî no scroll view. All zones are fixed height except `Spacer`. The screen does not scroll.

## Zone 1: ScoreBar

**Purpose:** Running score vs par. Always visible. Never truncated. Survives hole transitions.

**Dimensions:** Full width, height 56dp, `BrdyColors.surface` background.

**Layout:** `Row` with `MainAxisAlignment.spaceBetween`, horizontal padding `BrdySpacing.md` each side.

**Left element ‚Äî label:**
- Copy: `"SCORE"` (all caps, ASCII)
- Style: `helperText` (Barlow Condensed 14sp, weight 400)
- Color: `BrdyColors.onSurfaceMuted`

**Right element ‚Äî value:**
- Format rules (exact, no exceptions):
  - Score == 0: `"E"`
  - Score > 0: `"+{n}"` (ASCII plus sign, no space)
  - Score < 0: `"-{n}"` (ASCII hyphen-minus U+002D, not Unicode minus U+2212, no space)
- Style: `displayNumeric` (JetBrains Mono 28sp, weight 700)
- Color:
  - Score == 0 or Score > 0: `BrdyColors.onSurface`
  - Score < 0: `BrdyColors.accent`
- Initial state (no holes scored): show `"E"` in `BrdyColors.onSurface`

**Loading state:** Show `"‚Äì"` in `BrdyColors.onSurfaceMuted` while `runningScoreProvider` is in loading state.

## Zone 2: HoleHeader

**Purpose:** Hole number, par, Stroke Index. Prominent at-a-glance data a golfer needs before selecting outcome.

**Dimensions:** Full width, height 56dp, `BrdyColors.surface` background.

**Layout:** `Row` with `MainAxisAlignment.spaceBetween`, horizontal padding `BrdySpacing.md` each side. Three equally-spaced groups.

**Left group ‚Äî hole number:**
- Label: `"HOLE"` in `helperText` style, `BrdyColors.onSurfaceMuted`, above the value
- Value: `"{n}"` (1‚Äì18, no leading zero) in `bodyNumeric` style (JetBrains Mono 16sp), `BrdyColors.onSurface`

**Center group ‚Äî par:**
- Label: `"PAR"` in `helperText` style, `BrdyColors.onSurfaceMuted`, above the value
- Value: `"{n}"` (3, 4, or 5) in `bodyNumeric` style, `BrdyColors.onSurface`

**Right group ‚Äî stroke index:**
- Label: `"SI"` in `helperText` style, `BrdyColors.onSurfaceMuted`, above the value
- Value when available: `"{n}"` (1‚Äì18) in `bodyNumeric` style, `BrdyColors.onSurface`
- Value when null: `"‚Äì"` (en dash U+2013) in `bodyNumeric` style, `BrdyColors.onSurfaceMuted`
- Do not show `"N/A"`, `"?"`, or any other substitute ‚Äî use `"‚Äì"` only

**Loading state:** Show all three values as `"‚Äì"` in `BrdyColors.onSurfaceMuted` while `courseForRoundProvider` loads.

## Zone 3: Secondary Controls

All three controls share a `Column` with `BrdySpacing.sm` (8dp) vertical gap between each row. Horizontal padding: `BrdySpacing.md` (16dp) each side. Background: `BrdyColors.background` (not surface ‚Äî these float on the main background).

### PuttsCounter (SHOT-05)

**Always shown** ‚Äî regardless of hole par or scored state.

**Dimensions:** Full width row, height 48dp.

**Layout:** `Row` ‚Äî label left, counter group right.

**Label:**
- Copy: `"PUTTS"` (all caps, ASCII)
- Style: `helperText` (Barlow Condensed 14sp, weight 400)
- Color: `BrdyColors.onSurfaceMuted`

**Counter group (right-aligned):**
- Minus button: `"‚àí"` (minus sign, ASCII hyphen-minus U+002D)
  - `IconButton` or `TextButton`, minimum tap target 48√ó48dp
  - Disabled (not hidden) when `putts == 0`; `color: BrdyColors.onSurfaceMuted` when disabled
  - Active color: `BrdyColors.onSurface`
- Count value: `"{n}"` in `bodyNumeric` style (JetBrains Mono 16sp), `BrdyColors.onSurface`
  - Width: fixed 32dp minimum to prevent layout shift at double digits
  - Initial state: `"0"`
- Plus button: `"+"` (ASCII plus sign)
  - Always enabled (no upper limit enforced)
  - Active color: `BrdyColors.onSurface`

**Inactive state (outcome not yet recorded):** Entire row rendered at `opacity: 0.4`. Tapping minus/plus is a no-op (guarded by `HoleScoreNotifier.setPutts` null check ‚Äî no outcome row = no write). Do not hide the counter; show it dimmed to communicate it exists but requires outcome first.

### FairwayHitToggle (SHOT-06)

**Conditionally rendered:** Absent from widget tree entirely on par 3 holes. Use `if (par != 3)` inline in the `Column` children list ‚Äî not `Visibility` or `Opacity`. The widget must not exist in the tree on par 3 holes.

**Dimensions:** Full width row, height 48dp. Only rendered on par 4/5 holes.

**Layout:** `Row` ‚Äî label left, toggle right.

**Label:**
- Copy: `"FAIRWAY HIT"` (all caps, ASCII, two words)
- Style: `helperText` (Barlow Condensed 14sp, weight 400)
- Color: `BrdyColors.onSurfaceMuted`

**Toggle states:**
- Off state (missed / false): copy `"MISS"`, background `BrdyColors.surface`, text color `BrdyColors.onSurface`
- On state (hit / true): copy `"HIT"`, background `BrdyColors.accent`, text color `BrdyColors.onAccent`
- Style for state label: `buttonLabel` (Barlow Condensed 18sp, weight 700)
- Toggle widget: `TextButton` with fixed width 72dp and height 40dp ‚Äî NOT a `Switch` widget (too fine-motor for on-course)

**Inactive state (outcome not yet recorded):** Entire row at `opacity: 0.4`, taps are no-ops.

### GirToggle (SHOT-07)

**Always shown** ‚Äî regardless of hole par or scored state.

**Dimensions:** Full width row, height 48dp.

**Layout:** `Row` ‚Äî label left, toggle right.

**Label:**
- Copy: `"GIR"` (all caps, ASCII, no expansion)
- Style: `helperText` (Barlow Condensed 14sp, weight 400)
- Color: `BrdyColors.onSurfaceMuted`

**Toggle states:**
- Off state (missed / false): copy `"NO GIR"`, background `BrdyColors.surface`, text color `BrdyColors.onSurface`
- On state (green in regulation / true): copy `"GIR"`, background `BrdyColors.accent`, text color `BrdyColors.onAccent`
- Style for state label: `buttonLabel` (Barlow Condensed 18sp, weight 700)
- Toggle widget: `TextButton` with fixed width 72dp and height 40dp

**Inactive state (outcome not yet recorded):** Entire row at `opacity: 0.4`, taps are no-ops.

## Zone 4: OutcomeButtonGrid

**Purpose:** Primary action zone. One tap records the hole outcome and auto-advances.

**Dimensions:**
- Row 1: 80dp tall, full width (three buttons via `Expanded` flex)
- Gap between rows: `BrdySpacing.sm` (8dp)
- Row 2: 80dp tall, full width (two buttons via `Expanded` flex)
- Total zone height: 168dp

**Border radius:** 0dp on all buttons. No rounding. No shadows. No elevation.

**All buttons ‚Äî shared properties:**
- `BoxConstraints(minHeight: 80)` enforced via `SizedBox(height: 80)` wrapper around each button
- Label style: `buttonLabel` (Barlow Condensed 18sp, weight 700)
- Tap feedback: `HapticFeedback.mediumImpact()` on every outcome tap including EAGLE
- Press animation: scale 1.0 ‚Üí 0.96 ‚Üí 1.0 over 160ms total (80ms down, 80ms up) via `flutter_animate`

**Button specifications:**

| Button | Row | Copy | Background | Text Color | Gesture |
|--------|-----|------|------------|------------|---------|
| BIRDIE | 1, position 1 | `"BIRDIE"` | `BrdyColors.accent` | `BrdyColors.onAccent` | `GestureDetector` with `onTap` (single = BIRDIE) and `onDoubleTap` (double = EAGLE) |
| PAR | 1, position 2 | `"PAR"` | `BrdyColors.surface` | `BrdyColors.onSurface` | `InkWell.onTap` |
| BOGEY | 1, position 3 | `"BOGEY"` | `BrdyColors.surface` | `BrdyColors.onSurface` | `InkWell.onTap` |
| DOUBLE | 2, position 1 | `"DOUBLE"` | `BrdyColors.destructive` | `BrdyColors.onDestructive` | `InkWell.onTap` |
| PICKUP | 2, position 2 | `"PICKUP"` | `BrdyColors.destructive` | `BrdyColors.onDestructive` | `InkWell.onTap` |

**EAGLE double-tap specifics:**
- `GestureDetector` wraps the BIRDIE button widget ONLY ‚Äî not any parent container
- Single-tap on BIRDIE has ~300ms delay (Flutter gesture arena waiting for second tap) ‚Äî this is accepted; golfers deliberate before tapping
- EAGLE has no dedicated button; the button label does not change to show double-tap hint
- EAGLE haptic: same `HapticFeedback.mediumImpact()` as all outcomes
- EAGLE toast: `"EAGLE ‚Äî Hole {n}"` (same format as all outcomes)

**PAR/BOGEY button border:** `Border.all(color: BrdyColors.divider, width: 1)` to distinguish from background. BIRDIE, DOUBLE, PICKUP have their own saturated backgrounds and need no border.

**Disabled state:** None. All six outcomes are always tappable regardless of current hole state or previous entries. Re-tapping an already-scored hole overwrites the outcome (navigating back is how this happens ‚Äî see Zone 5).

## Undo Toast (SHOT-04)

**Mechanism:** `ScaffoldMessenger.showSnackBar` ‚Äî not a custom overlay.

**Trigger:** Fired immediately after `recordOutcome()` completes and `activeHoleIndexProvider` advances.

**Duration:** 4 seconds exactly (`duration: const Duration(seconds: 4)`).

**`persist: false`:** Mandatory. Flutter 3.38+ changed SnackBar with action to persist indefinitely by default. `persist: false` restores 4-second auto-dismiss.

**Content format (exact copy):**
- `"{OUTCOME_LABEL} ‚Äî Hole {n}"` where:
  - `OUTCOME_LABEL` is the ALL CAPS display string (see table below)
  - `{n}` is the 1-based hole number that was just scored
  - Separator is ` ‚Äî ` (space, en dash U+2013, space)

| HoleOutcome enum | OUTCOME_LABEL in toast |
|-----------------|----------------------|
| `eagle` | `"EAGLE"` |
| `birdie` | `"BIRDIE"` |
| `par` | `"PAR"` |
| `bogey` | `"BOGEY"` |
| `doubleBogey` | `"DOUBLE"` |
| `pickup` | `"PICKUP"` |

**Examples:**
- `"BIRDIE ‚Äî Hole 7"`
- `"DOUBLE ‚Äî Hole 14"`
- `"EAGLE ‚Äî Hole 3"`

**SnackBar visual:**
- `backgroundColor: BrdyColors.surface`
- `content` text style: `helperText` (Barlow Condensed 14sp), color `BrdyColors.onSurface`

**Undo action:**
- `SnackBarAction` label: `"UNDO"` (all caps, ASCII)
- `textColor: BrdyColors.accent`
- `onPressed` sequence:
  1. `HapticFeedback.lightImpact()`
  2. `holeScoreNotifierProvider(roundId, scoredHoleIndex).notifier.undoOutcome()`
  3. `ref.read(activeHoleIndexProvider.notifier).set(scoredHoleIndex)` ‚Äî returns to the just-undone hole

**Queue behavior:** `ScaffoldMessenger.of(context)..hideCurrentSnackBar()..showSnackBar(...)` ‚Äî each new outcome dismisses any prior toast before showing the new one.

**Position:** Material default (bottom of screen, above `HoleNavDrawer`). The `HoleNavDrawer` is inside the `Scaffold` body column, not a `bottomNavigationBar`, so the snackbar renders above the drawer naturally.

## Hole Navigation Interaction Model (SHOT-12)

This is NOT router navigation. All hole switching is `ref.read(activeHoleIndexProvider.notifier).set(targetIndex)`.

**Back navigation (correcting a past hole):**
1. User taps a scored chip in HoleNavDrawer
2. `activeHoleIndexProvider` updates to `targetIndex`
3. `ShotCaptureScreen` re-renders with data for that hole (from `holeScoreNotifierProvider(roundId, targetIndex)`)
4. All outcome buttons remain active ‚Äî tapping records a corrected outcome (overwrites Drift row)
5. `"NOW"` button appears in HoleNavDrawer (since `activeHoleIndex < highestScoredHoleIndex`)

**Correction flow visual distinction:**
- "Previously scored" hole: HoleNavDrawer chip shows that hole's outcome color with active border (width 2, `BrdyColors.onSurface`)
- Outcome buttons: no visual change ‚Äî they look identical to the live-scoring state
- No "editing" banner or indicator ‚Äî the chip color + active border is sufficient context

**Forward navigation (impossible to skip ahead):**
- Future holes (beyond `highestScoredHoleIndex + 1`) have `onTap: null` and `opacity: 0.4`
- Auto-advance after outcome tap is the only way to reach a new hole

**Round completion:**
- After scoring hole 18 (index 17), `roundCompleteProvider` returns `true`
- `ref.listen` in screen triggers: `roundDao.completeRound(roundId, DateTime.now())` + `context.go('/round-review/$roundId')`
- No confirmation dialog ‚Äî completion is automatic and immediate
