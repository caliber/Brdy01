  <name>Task 2: Implement watch-side WearDataBridgePlugin (Kotlin)</name>
  <files>watch/android/app/src/main/kotlin/com/brdy/brdy01/WearDataBridgePlugin.kt, watch/android/app/src/main/kotlin/com/brdy/brdy01/MainActivity.kt</files>
  <action>
    Create `watch/android/app/src/main/kotlin/com/brdy/brdy01/WearDataBridgePlugin.kt`. Package: `com.brdy.brdy01`.

    The class implements `FlutterPlugin`, `MethodCallHandler`, and `DataClient.OnDataChangedListener`. It manages TWO EventChannels (hole + connection) using two inner StreamHandler instances.

    Required behavior:
    - `onAttachedToEngine`: cache context. Create `MethodChannel(messenger, "com.brdy.brdy01/wear")` -> `setMethodCallHandler(this)`. Create `EventChannel(messenger, "com.brdy.brdy01/wear/hole")` and assign a StreamHandler that stores `holeSink`. Create `EventChannel(messenger, "com.brdy.brdy01/wear/connection")` and assign a StreamHandler that stores `connSink` AND starts a polling job (use `kotlinx.coroutines` with `Dispatchers.Main` or a `Handler` that posts every 3000 ms) that calls `Wearable.getNodeClient(context).connectedNodes` and pushes `nodes.isNotEmpty()` to `connSink` whenever the value changes. Stop the job in `onCancel`. Add the listener with `Wearable.getDataClient(context).addListener(this)`.
    - `onMethodCall`:
      - `"sendScore"`: args `{ holeNumber: Int, outcome: String, timestamp: Long }`. Build `PutDataMapRequest.create("/brdy/score/$holeNumber-$timestamp")`. Put `holeIndex` (use `holeNumber - 1`? — NO, mirror the field used by phone: store both `holeNumber` and `holeIndex = holeNumber - 1` to avoid ambiguity; phone reads `holeIndex`). Put `outcome` (string), `ts` (timestamp). Call `.asPutDataRequest().setUrgent()`. `Wearable.getDataClient(context).putDataItem(request).addOnSuccessListener { result.success(null) }.addOnFailureListener { result.error("PUT_FAILED", it.message, null) }`.
      - `"connectionStatus"`: query `Wearable.getNodeClient(context).connectedNodes`. Return bool.
      - default `result.notImplemented()`.
    - `onDataChanged`: iterate events. For each `TYPE_CHANGED` event with `uri.path == "/brdy/hole"`, decode `DataMapItem.fromDataItem(event.dataItem).dataMap`. Read `roundId`, `holeIndex`, `par`, `si` (nullable — `if (dataMap.containsKey("si")) dataMap.getInt("si") else null`), `ts`. Post to main thread and call `holeSink?.success(mapOf("roundId" to roundId, "holeIndex" to holeIndex, "par" to par, "si" to si, "ts" to ts))`. Ignore `/brdy/ack/...` events (or pass-through via separate event in a future plan).
    - `onDetachedFromEngine`: remove listener, cancel polling, null channels.

    Update `watch/android/app/src/main/kotlin/com/brdy/brdy01/MainActivity.kt`: extend `FlutterActivity`, override `configureFlutterEngine`, add `WearDataBridgePlugin()` to `flutterEngine.plugins`.

    Imports: same as Plan 04-01 plus `kotlinx.coroutines.*` only if coroutines used; otherwise `android.os.Handler`/`Looper` for polling.

    Use DataClient ONLY. No MessageClient.
  </action>
