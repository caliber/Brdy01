## Decisions Made

- `readOnly` parameter is NOT passed in the `/round-review/:roundId` route builder yet. A TODO comment marks the spot. Wave 3 (06-03) will add the `readOnly` constructor parameter to `RoundReviewScreen` and then the router line can be activated. This avoids a compile error in this wave while documenting the dependency clearly.
- The redirect allowlist uses two separate conditions: exact path `== '/round-history'` and `startsWith('/round-review/')` with `queryParameters['readOnly'] == 'true'`. This is the minimum allowlist needed per T-06-04 mitigation.
- `context.push('/round-history')` (not `context.go`) is used from SetupScreen to preserve Setup in the back-stack per CLAUDE.md navigation model.

## Deviations from Plan

None - plan executed exactly as written. The readOnly Wave 3 dependency was pre-documented in the plan itself, and the TODO comment approach was explicitly suggested in the plan action for Task 2.

## Threat Mitigations Applied

- **T-06-03 (Tampering — Dismissible fires without confirmation):** `confirmDismiss` callback gates `onDismissed`; the `?? false` guard handles dialog-dismissed-without-choice (back tap, tap outside dialog).
- **T-06-04 (DoS — redirect infinite loop):** `state.uri.path == '/round-history'` check returns `null` before the shot-capture redirect fallback. Verified in router.dart line 56.

## Known Stubs

None. `completedRoundsProvider` is wired to the live DAO stream. The tile reads stats from `statsProvider`. The readOnly navigation path is fully wired on the tile side — only the screen constructor parameter is deferred to Wave 3.
