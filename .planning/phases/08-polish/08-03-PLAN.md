Create `lib/features/shot_capture/widgets/mini_scorecard_overlay.dart` as a `ConsumerWidget`.

Constructor params: `roundId` (int, required), `isOpen` (bool, required).

The widget reads:
- `ref.watch(holeListProvider(roundId))` for hole outcomes
- `ref.watch(activeHoleIndexProvider)` for the current active hole
- `ref.watch(highestScoredHoleIndexProvider(roundId))` for which holes are tappable
- `ref.watch(runningScoreProvider(roundId))` for the score summary line

Structure:
```
AnimatedContainer(
  duration: 200ms, curve: Curves.easeOut,
  height: isOpen ? 132 : 0,
  color: BrdyColors.background.withOpacity(0.92),
  child: isOpen ? _buildContent(...) : SizedBox.shrink(),
)
```

`_buildContent` renders:
1. Row 1 of 9 chips — holes 1–9 (holeNumber 1..9, index 0..8)
2. Row 2 of 9 chips — holes 10–18 (holeNumber 10..18, index 9..17)
3. Score summary bar: a Text showing the running score as `'+N'`, `'-N'`, or `'E'` (even), with `SometypeMono` 14sp font, colour white

Each chip:
- Size: 40×40 with 4dp BorderRadius (matches `_HoleChip` in `hole_nav_drawer.dart`)
- Fill: `HoleNavDrawer._outcomeToFill(outcomeName)` — reuse the static method directly
- Text colour: `HoleNavDrawer._outcomeToTextColor(outcomeName)`
- Content: hole number as string, 14sp SometypeMono bold
- Active border: `Border.all(color: BrdyColors.onSurface, width: 2)` when `index == activeHoleIndex`
- onTap: tappable if `index <= highestScoredIndex + 1`; on tap:
  `await HapticFeedback.lightImpact(); ref.read(activeHoleIndexProvider.notifier).set(index);`
  Note: closing the overlay on chip tap is handled in the screen (the screen will pass an `onChipTapped` callback — see Task 2). Alternatively wire the close by passing a `VoidCallback? onClose` param and calling it after setting the index. Use `onClose` callback (preferred — keeps overlay widget stateless).

Add `onClose` as an optional `VoidCallback?` to the constructor. Call `onClose?.call()` inside each chip's onTap after `activeHoleIndexProvider.notifier.set(index)`.

Bogey text decoration: apply `TextDecoration.underline` on the chip number if the outcome is `HoleOutcome.bogey` (same as `_HoleChip` in `hole_nav_drawer.dart`).

Imports needed: flutter/material.dart, flutter/services.dart, flutter_riverpod, google_fonts, hole_nav_drawer.dart (for the static colour methods), active_hole_index_provider.dart, highest_scored_hole_index_provider.dart, hole_list_provider.dart, running_score_provider.dart, hole_outcome.dart, brdy_colors.dart, brdy_spacing.dart.

IMPORTANT design constraints (from RESEARCH.md):
- Do NOT use `GlobalKey` or `OverlayEntry` — plain `AnimatedContainer` only
- Do NOT expand `_TopZone` height — 132dp fits in 36% zone even on smallest device
- WCAG AAA contrast: all text in the overlay must use tokens from `BrdyColors` — no off-palette colours
